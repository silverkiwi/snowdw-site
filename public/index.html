// How it works

From messy Postgres to analytics-ready warehouse

Five guided steps. No manual schema design. No consulting engagement. No six-month project.

01

Profile your tables

SnowDW scans every table in your Postgres schema: row counts, column cardinality, sample values, data types, and inferred dimension candidates. Works with any Postgres instance — RDS, Supabase, Neon, Railway, self-hosted.

02

Infer foreign key relationships

SnowDW finds relationships between tables by comparing distinct value overlaps — even when FKs aren't formally declared. In a real run it found 101 relationships with high confidence, including ones the original schema didn't document.

03

AI designs your star schema

Choose any LLM via OpenRouter (Deepseek, Claude, GPT-4o, etc.) or bring your own Claude API key. The model designs a Kimball-style star schema — fact tables, dimension tables, grains — and explains its reasoning. Not happy with the output? Just tell it what to change in plain English and it redesigns on the spot.

04

Generate the dbt project

Approve the schema design and SnowDW generates a complete dbt project: sources.yml, staging models, mart models, and schema docs. Standard, readable SQL — your codebase to own and extend.

05

Run dbt — green on first try

Hit "Run dbt" directly in the UI. SnowDW runs dbt deps → dbt run → dbt test and streams the output live. In a real run on 7 source tables: PASS=22 WARN=0 ERROR=0 in under 6 seconds.
