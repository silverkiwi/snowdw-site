// How it works

From messy Postgres to analytics-ready warehouse

Four steps. No manual schema design. No consulting engagement. No six-month project.

01

Connect your Postgres database

Point SnowDW at your database with a connection string. It works with any Postgres instance — RDS, Supabase, Neon, Railway, self-hosted.

02

Automatic table profiling

SnowDW scans every table: row counts, column cardinality, sample values, data types. It infers foreign key relationships even if they're not formally declared. No config required.

03

LLM-designed star schema

The profiling context is sent to an LLM (Claude or DeepSeek via OpenRouter) which designs a Kimball-style star schema: fact tables, dimension tables, and grain definitions. It reasons about your actual data, not a generic template.

04

Complete dbt project output

SnowDW generates a full dbt project: sources.yml, staging models, mart models, and schema docs. Run dbt run and your warehouse is live.
